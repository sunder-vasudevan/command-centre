🤖 Agent Protocol — How Claude behaves on every task

📋 Edit Approval Gate

  • States files + changes before touching anything
  • Waits for "go ahead / yes / do it"
  • Exception: wrap/session tasks run autonomously
  • Never changes permission mode without approval

🗺️ Planning

  • Plan mode for 3+ step tasks or arch decisions
  • Goes sideways? Stop and re-plan — don't push
  • Specs written upfront before any code

🔀 Subagents

  • Used liberally to keep main context clean
  • Research, exploration, parallel work → offloaded
  • One focused task per subagent

✅ Verification

  • Never marked done without proof it works
  • "Would a staff engineer approve this?" — always asked

💎 Elegance

  • Non-trivial: "is there a more elegant way?"
  • Simple fixes: skip — don't over-engineer

🐛 Bug Reports

  • Given a bug: just fix it — no hand-holding
  • Logs → root cause → fix → report back
🔬 Strip all assumptions before building

Say "first principles" or "break it down" before starting any new domain, project, or complex decision. Claude picks the right mode and strips inherited thinking before planning begins.

Mode Best for What it does
Feynman Understanding a concept for the first time Explain as if I'm 12. No jargon. Keep going until genuinely simple.
Learning Entering a new field Break down the 5 core assumptions beginners just accept as true.
First Principles Any topic — strip inherited thinking Strip every assumption. Rebuild from what's provably true.
Business Product / business problem List every assumption. Find the most direct path to outcome.
Musk Test Rethinking an existing solution Starting from zero — how would you build this differently today?
Full Stack New project, zero expertise Runs all 5 modes in order before any planning starts.

🔄 How a Session Actually Runs

Based on real session patterns from ARIA (9 sessions), BzHub (4 sessions), and meta sessions. Every session follows this shape — regardless of the trigger word used.

▶
Phase 1 — Start
🏎️
Say the trigger word Sunny
lights out → Claude reads NOTES.md, last commit, parking lot — briefing given  ·  fury road → skip to scope  ·  cold start → full audit before anything
🤝
Scope locked in one sentence Both
No code touched until the task is a single sentence. New ideas → project_parking_lot.md, not this session.
▶
Phase 2 — Build Loop
⚙️
Diagnose first, code second Claude
Reads clients.py, models.py, routes/*.py, component files — root cause identified before any edit is made.
🤝
Edit gate — state files, get approval Both
Claude: "I'll edit X, Y, Z — here's why." Sunny: "go ahead." No surprises. Wrap-time docs (NOTES.md, logs) run autonomously — no gate needed.
⚙️
Subagents run in parallel Claude
ARIA Session 9: Kanban/Funnel built by background subagent while CRM table shipped in main context. BzHub: whitepapers + decks generated in background during wrap. Same output, half the clock time.
🏎️
Redirect loop Sunny
Sunny spots something → "same issue, try this" or "that's not the right look" → Claude adapts without re-briefing. Safari date bug took 3 rounds before the month/year select fix landed. That's normal — the loop is fast.
🤝
New idea mid-session → park it Both
"park that" saves to project_parking_lot.md. Claude flags scope creep when a new feature arrives before the committed one is done. Mandatory pushback — not optional.
▶
Phase 3 — Wrap
⚙️
DoD checklist Claude
Works end-to-end, mobile tested, no errors, auth respected → definition_of_done.md
⚙️
Docs updated Claude
NOTES.md · DECISION_LOG.md · documentation/INTERACTION_LOG.md · ~/claude-memory/EFFICIENCY_LOG.md · memory files
🤝
git commit → push → /compact Both
Clear commit message, no Co-Authored-By → sync.sh push → /compact — session closed cleanly 🏁

📄 Claude's permanent standing orders — loaded every conversation, every project

Think of CLAUDE.md as the rulebook baked into every session. You never need to repeat these rules — they're always active from the moment you start talking.

📍
Where it lives
~/.claude/CLAUDE.md — global file, applies to every project. Projects can layer their own CLAUDE.md on top.
🔁
Always loaded
Unlike memory files (loaded selectively), CLAUDE.md is always in context. Every conversation starts with these rules already active.
⚡
What's in yours right now
Standing rules (no Co-Author tag) · Agent Protocol (edit gate, planning, subagents, verification, elegance, bugs) · gstack skills list · Session start time rule
🧠
Memory files vs CLAUDE.md
Memory files store project state and feedback — loaded when relevant. CLAUDE.md stores rules that must always be active regardless of project.
🪝
Hooks (settings.json)
Hooks in ~/.claude/settings.json fire when you say trigger words (wrap, lights out, first principles). They inject the trigger file instructions before Claude reads your message — that's how one word kicks off a full protocol.
✏️
When to update it
When a rule should apply to every project forever. Project-specific things go in project memory files or a project-level CLAUDE.md.
