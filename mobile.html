Protocol
Agent Protocol
6 rules — edit gate, planning, subagents, verify, elegance, bugs
  1. Edit Approval Gate — State files + changes upfront. Wait for explicit "go ahead" before editing. Exception: wrap/session tasks run autonomously.
  2. Planning — Enter plan mode for 3+ step tasks or architectural decisions. Write specs upfront. If something goes sideways mid-task, stop and re-plan.
  3. Subagents — Use subagents liberally to keep main context clean. Offload research, exploration, and parallel analysis. One focused task per subagent.
  4. Verification — Never mark a task complete without proving it works. Ask: "Would a staff engineer approve this?" before presenting.
  5. Elegance — For non-trivial changes, pause and ask: "Is there a more elegant way?" Skip for simple, obvious fixes.
  6. Bug Reports — When given a bug: fix it. Don't ask for hand-holding. Point at logs/errors/tests, resolve them, report back.

How a Session Runs
Phase 1 — Start
4 steps to orient and align before building
  1. Capture session start time via date command — log internally
  2. Load CLAUDE.md + MEMORY.md — rehydrate project state
  3. Review parking lot from last session — surface any captured ideas
  4. Confirm today's goal with Sunny — one clear outcome per session
Phase 2 — Build Loop
5-step cycle: plan → gate → build → verify → compact
  1. State plan upfront — files, changes, approach — wait for gate approval
  2. Execute in focused increments — one feature unit at a time
  3. Prove it works before calling it done — logs, tests, browser check
  4. Flag red flags immediately — security, cost, breaking changes surface fast
  5. Run /compact when context gets heavy — snapshot first, then compress
Phase 3 — Wrap
4 steps to close cleanly and preserve session value
  1. Update MEMORY.md — project state, learnings, next actions
  2. Write context snapshot if session was complex or multi-file
  3. Log PO time and compression estimate to session log
  4. Confirm parking lot is clear — any ideas captured mid-session noted
